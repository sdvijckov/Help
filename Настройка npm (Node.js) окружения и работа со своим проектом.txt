# Настройка Node.js (npm) окружения и работа со своим проектом

Убедиться, что Node.js и npm установлены и добавлены в PATH. Проверить это можно, открыв командную строку и введя:
```bash
node -v
npm -v
```

Создать структуру проекта (обычно с помощью специальных инструментов, например, `npx create-react-app`, или вручную, как мы делали).

### Работа с зависимостями проекта (локально)

**1. Перейти в папку проекта:**
```bash
D:
cd путь\к\папке\проекта
```

**2. Инициализировать проект (если еще не сделано и нет package.json):**
Это создаст файл `package.json` с базовыми метаданными.
```bash
npm init -y
```
Дополнительно активировать не требуется, при входе в папку это происходит автоматически.
**3. Установить все зависимости из package.json:**
Эта команда прочитает файл `package.json` и установит все пакеты (из `dependencies` и `devDependencies`) в папку `node_modules` внутри твоего проекта. Это и есть аналог "виртуального окружения" в Node.js — каждый проект изолирован по умолчанию.
```bash
npm install
```

**4. Установить новую библиотеку:**
*   Для обычной зависимости (нужна для работы приложения):
    ```bash
    npm install название_библиотеки
    ```
    или сокращенно:
    ```bash
    npm i название_библиотеки
    ```
*   Для зависимости разработки (нужна только во время разработки, например, для тестирования или сборки):
    ```bash
    npm install --save-dev название_библиотеки
    ```
    или сокращенно:
    ```bash
    npm i -D название_библиотеки
    ```

**5. Удалить библиотеку:**
```bash
npm uninstall название_библиотеки
```
Удалить окружение:
```bash
войти в папку проекта 
rmdir /s /q node_modules

**6. Проверить установленные пакеты:**
Все установленные пакеты будут находиться в папке `node_modules`. Вы также можете просмотреть список установленных локальных пакетов:
```bash
npm list --depth=0
```
Для просмотра всех пакетов, включая подзависимости:
```bash
npm list
```

**7. Запустить скрипты проекта:**
В файле `package.json` в секции `"scripts"` могут быть определены команды для запуска, сборки, тестирования и других задач. Например, если в `package.json` есть:
```json
"scripts": {
  "start": "node index.js",
  "test": "jest"
}
```
То можно запустить их так:
```bash
npm run start
# или для скриптов "start", "test", "stop", "restart" можно использовать сокращенную форму:
npm start

npm run test
# или
npm test
```

### Глобальная установка пакетов (для утилит командной строки)

Иногда вам могут понадобиться инструменты, которые доступны из любой директории. Их устанавливают глобально:
```bash
npm install -g название_пакета
```
Пример: установка `live-server` (сервер для разработки, который мы добавляли в некоторые `package.json`):
```bash
npm install -g live-server
```
Однако для зависимостей самого проекта всегда предпочительна локальная установка (`npm install` без `-g`).

### Проблемы с PATH

Если после установки Node.js или глобальных пакетов команды `node`, `npm`, `yarn` или другие глобальные инструменты недоступны, убедитесь, что путь к Node.js и глобальным пакетам npm добавлен в системную переменную среды `PATH` в Windows. Обычно это делается автоматически при установке Node.js, но иногда может потребовать ручной проверки.
