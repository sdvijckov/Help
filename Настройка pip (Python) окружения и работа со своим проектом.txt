Создать репозиторий в GitHub Desktop в разделе D.
Убедиться, что Python установлен и добавлен в PATH. Проверить это можно, открыв командную строку и введя:
python --version
pip --version

Создать структуру проекта (можно попросить любую нейросеть написать скрипт для создания структуры в папке проекта).
Скрипт запускается из командной строки в папке, где лежит файл:
python название_скрипта.py

Создать своё виртуальное окружение:
Перейти в папку проекта:
D:
cd путь\к\папке\проекта
Создать окружение:
python -m venv название_проекта_env

Активировать окружение (нужно делать каждый раз перед работой с проектом):
.\название_проекта_env\Scripts\activate

После активации окружения в начале командной строки появится (название_проекта_env).
Проверить версию Python и установленные пакеты:
python --version

Убедитесь, что там есть базовый пакет pip;
pip list
И обновите, если нужно:
python.exe -m pip install --upgrade pip

Если нужно установить библиотеку:
pip install название_библиотеки
или из файла
pip install -r requirements.txt
Проверить совместимость пакетов:
pip check

Проверка, что библиотека установилась:
pip show название_библиотеки
Или в коде:
import название_библиотеки
print("Библиотека работает!")

Собрать информацию об окружении в файл:
Выполнить команду в активированном окружении:
pip freeze > requirements.txt

Если нужно удалить или пересоздать окружение:
Деактивировать окружение:
deactivate
Затем просто удалить папку с окружением:
rmdir /s /q название_проекта_env

Создать окружение заново:
python -m venv название_проекта_env
Активировать его:
.\название_проекта_env\Scripts\activate

Чтобы работать с кодом и запускать его, нужно:
Активировать окружение через терминал:
.\название_проекта_env\Scripts\activate
Перейти в папку с файлом:
D:
cd Путь/к_папке/проекта/src/Папка_где_лежит_файл
Создать файл:
type nul > Название_файла.py
Открыть файл в редакторе:
start notepad++.exe Название_файла.py

Запустить файл:
python main.py

Восстановить окружение на другом компьютере:
Создать и активировать новое окружение:
python -m venv новое_название_env
.\новое_название_env\Scripts\activate
Установить пакеты из файла requirements.txt:
pip install -r requirements.txt

Чтобы установить новую библиотеку, не обязательно выходить из папки программы. Главное, чтобы окружение было активировано.
Чтобы выйти из текущей папки (подняться на уровень вверх), используйте команду:
cd ..
Чтобы создать новую папку, используйте команду:
mkdir название_папки

Чтобы собрать .exe файл, нужно:
Установить библиотеку в соответствующее окружение:
pip install pyinstaller
Перейти в папку проекта и выполнить команду:
pyinstaller --windowed main.py
Если есть иконка:
pyinstaller --windowed --icon="путь/к/вашей/myicon.ico" main.py

В папке проекта появятся две папки (build, dist) и файл .spec.
Папку build можно удалить после сборки.
Перед каждым новым билдом проекта рекомендуется удалять старые папки сборки:
rmdir /s /q build
rmdir /s /q dist
И пересобрать проект:
pyinstaller --windowed main.py
Для распространения нужно запаковать в архив папку с программой, которая находится в папке dist.

Проблемы с SSL-сертификатами:
Часто решаются обновлением связанных библиотек.
pip install --upgrade requests urllib3 pyopenssl cryptography idna certifi
Это может быть вызвано нестабильным интернет-соединением или проблемами на сервере.
