# Что значит "Запустить скрипты проекта"?

Этот раздел объясняет, что такое "скрипты" в файле `package.json` и как с ними работать.

В файле `package.json` есть специальная секция `"scripts"`. В этой секции разработчик может определить короткие, легко запоминающиеся команды (скрипты), которые на самом деле являются обёртками для более длинных или сложных команд.

### Для чего нужны скрипты в `package.json`?

1.  **Автоматизация:** Они позволяют автоматизировать повторяющиеся задачи, такие как запуск приложения, сборка проекта, тестирование, проверка стиля кода и т.д.
2.  **Удобство:** Вместо того чтобы запоминать длинные и сложные команды, вы просто запускаете короткий, осмысленный скрипт.
3.  **Унификация:** Все разработчики в команде запускают одни и те же скрипты, что обеспечивает единообразие в работе с проектом.
4.  **Изоляция:** Скрипты автоматически используют локально установленные пакеты из `node_modules`, даже если эти пакеты не установлены глобально.

### Как это выглядит в `package.json`?

Пример секции `scripts`:

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest",
    "build": "webpack --config webpack.config.js",
    "lint": "eslint . --ext .js,.jsx,.ts,.tsx"
  },
  "dependencies": {
    "express": "^4.17.1"
  },
  "devDependencies": {
    "jest": "^27.0.6",
    "nodemon": "^2.0.12",
    "webpack": "^5.0.0",
    "eslint": "^8.0.0"
  }
}
```

### Как запускать эти скрипты?

#### Для большинства скриптов

Используется команда `npm run <script-name>`.

Например:

```bash
# Запустит команду "nodemon index.js" из примера выше
npm run dev

# Запустит команду "webpack --config webpack.config.js"
npm run build

# Запустит команду "eslint . --ext .js,.jsx,.ts,.tsx"
npm run lint
```

#### Для стандартных скриптов

Для некоторых стандартных скриптов (`start`, `test`, `stop`, `restart`) можно использовать сокращенную форму без `run`.

Например:

```bash
# Запустит команду "node index.js" из примера выше
npm start

# Запустит команду "jest"
npm test
```

### В чем отличие от Python?

В Python вы обычно запускаете скрипты напрямую (`python main.py`) или как модули (`python -m my_module`).

В Node.js через `npm run <script-name>` вы централизуете все действия с проектом в одном месте (`package.json`), делая их более стандартизированными и легкодоступными. Это позволяет разработчику не знать всех тонкостей (например, как именно собирается проект), а просто выполнить команду `npm run build`.
