# 04_Управление инструментами и моделями

> Команды для управления инструментами, моделями ИИ и расширениями Qwen Code

---

## Список команд

| Команда | Назначение |
|---------|----------|
| [`/mcp`](#mcp) | Показать настроенные MCP серверы и инструменты |
| [`/tools`](#tools) | Показать список доступных инструментов |
| [`/skills`](#skills) | Показать и запустить доступные навыки |
| [`/model`](#model) | Переключить модель в текущей сессии |
| [`/extensions`](#extensions) | Показать активные расширения |
| [`/memory`](#memory) | Управление контекстом инструкций ИИ |

---

## `/mcp`

**Назначение:** Показывает настроенные MCP (Model Context Protocol) серверы и доступные инструменты.

**Когда использовать:**
- Хотите узнать, какие MCP серверы подключены
- Нужно проверить доступные инструменты от сторонних сервисов
- Настраиваете интеграцию с внешними API
- Диагностируете проблемы с инструментами

**Что делает:**
- Показывает список подключённых MCP серверов
- Отображает доступные инструменты от каждого сервера
- Показывает статус подключения к серверам

**Пример:**
```
/mcp
```

**Результат:**
```
Настроенные MCP серверы:

[✓] GitHub API
    - search_repositories
    - get_user_info
    - create_issue

[✓] File System
    - read_file
    - write_file
    - list_directory

[✗] Database (не подключён)
    - query_database
```

**С опцией `desc`:**
```
/mcp desc
```
Показывает подробное описание каждого инструмента.

## `/tools`

**Назначение:** Показывает список всех доступных инструментов в текущей сессии.

**Когда использовать:**
- Хотите увидеть все доступные инструменты
- Нужно проверить возможности Qwen в текущей сессии
- Настраиваете рабочий процесс
- Отлаживаете проблемы с инструментами

**Что делает:**
- Показывает полный список инструментов
- Группирует инструменты по категориям
- Показывает статус каждого инструмента

**Пример:**
```
/tools
```

**Результат:**
```
Доступные инструменты:

📁 Файловая система:
  - read_file
  - write_file
  - edit
  - list_directory

🌐 Web:
  - web_fetch
  - web_search

🔧 Код:
  - run_shell_command
  - task (субагенты)

📊 Навыки:
  - pdf
  - xlsx
```

**С опцией `desc`:**
```
/tools desc
```
Показывает подробное описание каждого инструмента с примерами использования.

## `/skills`

**Назначение:** Показывает доступные навыки (skills) и позволяет их запустить.

**Когда использовать:**
- Нужно обработать файлы специальных форматов (PDF, XLSX)
- Хотите использовать специализированный инструмент
- Работаете с конкретным типом данных
- Нужна специализированная обработка

**Что делает:**
- Показывает список установленных навыков
- Позволяет запустить выбранный навык
- Интегрирует специализированный функционал

**Пример:**
```
/skills
```

**Результат:**
```
Доступные навыки:

- pdf          — Работа с PDF файлами
- xlsx         — Работа с Excel таблицами
- image        — Обработка изображений
- audio        — Работа с аудио файлами

Введите название навыка для запуска:
```

**Запуск навыка:**
```
/skills pdf
```
Запускает навык работы с PDF.

**Важно:** Навыки должны быть установлены отдельно. По умолчанию могут быть недоступны.

## `/model`

**Назначение:** Переключает модель ИИ, используемую в текущей сессии.

**Когда использовать:**
- Хотите попробовать другую модель
- Нужна более быстрая/дешёвая модель для простых задач
- Требуется более мощная модель для сложных задач
- Тестируете разные модели

**Что делает:**
- Показывает список доступных моделей
- Позволяет выбрать модель для текущей сессии
- Применяет выбор немедленно

**Пример:**
```
/model
```

**Результат:**
```
Доступные модели:

[1] Qwen-Max      — Самая мощная (медленнее, дороже)
[2] Qwen-Plus     — Баланс скорости и качества
[3] Qwen-Turbo    — Быстрая и дешёвая (для простых задач)

Текущая модель: Qwen-Plus

Выберите модель (1-3):
```

**Типичные сценарии:**
- **Qwen-Turbo:** Быстрые вопросы, простой код, черновики
- **Qwen-Plus:** Повседневная разработка (рекомендуется по умолчанию)
- **Qwen-Max:** Сложные архитектурные задачи, отладка, рефакторинг

## `/extensions`

**Назначение:** Показывает все активные расширения в текущей сессии.

**Когда использовать:**
- Хотите проверить установленные расширения
- Нужно узнать, какие расширения влияют на работу
- Отлаживаете конфликты между расширениями
- Планируете установить новое расширение

**Что делает:**
- Показывает список активных расширений
- Отображает версию каждого расширения
- Показывает описание и автора

**Пример:**
```
/extensions
```

**Результат:**
```
Активные расширения:

[✓] Git Tools v1.2.0
    Автор: Qwen Team
    Интеграция с Git

[✓] Docker Helper v0.9.1
    Автор: Community
    Управление Docker контейнерами

[✓] API Tester v2.1.0
    Автор: Qwen Team
    Тестирование REST API
```

**Важно:** Расширения могут добавлять новые команды и инструменты.

## `/memory`

**Назначение:** Управляет контекстом инструкций и памяток для ИИ.

**Когда использовать:**
- Хотите сохранить важную информацию о проекте
- Нужно добавить контекст для будущих сессий
- Работаете с долгосрочным проектом
- Хотите персонализировать ответы Qwen

**Синтаксис:**
```
/memory add <текст>     # Добавить информацию в память
/memory list            # Показать сохранённую информацию
/memory clear           # Очистить память
/memory remove <ID>     # Удалить элемент по ID
```

**Примеры:**

**Добавление информации:**
```
/memory add Проект использует Python 3.11 и Flask
```

**Просмотр памяти:**
```
/memory list
```

**Результат:**
```
Сохранённая информация:

[1] Проект использует Python 3.11 и Flask
    Добавлено: 22.02.2026 14:30

[2] Предпочитаемый стиль кода: PEP 8
    Добавлено: 22.02.2026 15:15

[3] База данных: PostgreSQL 15
    Добавлено: 22.02.2026 16:00
```

**Удаление:**
```
/memory remove 1
```

**Очистка всей памяти:**
```
/memory clear
```

**Что сохраняется в памяти:**
- Информация о проекте
- Предпочтения по стилю кода
- Технические требования
- Персональные настройки
- Важные факты о пользователе

**Важно:** Память сохраняется между сессиями и проектами (глобальная память).

## Взаимосвязь команд

Эти команды часто используются вместе:

**Пример рабочего процесса:**

1. Проверить доступные инструменты:
   ```
   /tools
   ```

2. Проверить подключённые MCP серверы:
   ```
   /mcp
   ```

3. Добавить контекст о проекте:
   ```
   /memory add Проект: веб-скрапер на Python
   ```

4. Переключиться на более мощную модель для сложной задачи:
   ```
   /model
   # Выбрать Qwen-Max
   ```

5. Проверить активные расширения:
   ```
   /extensions
   ```

---

## Советы по использованию

1. **Регулярно проверяйте `/tools`** — узнавайте о новых возможностях.

2. **Используйте `/memory` для важного контекста** — экономит время на объяснениях.

3. **Переключайте `/model` по необходимости** — не используйте мощную модель для простых задач.

4. **Изучите `/mcp desc`** — подробные описания инструментов помогают открыть новые возможности.

5. **Следите за `/extensions`** — расширения могут добавлять полезные команды.
