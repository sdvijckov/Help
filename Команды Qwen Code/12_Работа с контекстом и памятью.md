## Что такое контекст?

**Контекст** — это вся информация, которую Qwen Code использует для понимания вашей задачи и генерации ответов.

### Компоненты контекста:

| Компонент | Описание | Тратит токены? |
|-----------|----------|----------------|
| **История чата** | Все сообщения в текущей сессии | ✅ Да |
| **Контекстные файлы** | `QWEN.md`, инструкции, настройки | ❌ Нет* |
| **Открытые файлы (`@`)** | Содержимое файлов, которые вы открыли | ✅ Да |
| **Системные инструкции** | Глобальные настройки и правила | ❌ Нет* |
| **Результаты инструментов** | Вывод команд, чтение файлов | ✅ Да |

*\* Контекстные файлы загружаются в системный контекст и не учитываются в лимите сессии*

## Контекстное окно и лимиты

### Лимиты токенов по моделям

| Модель | Входные токены | Выходные токены | Рекомендация |
|--------|----------------|-----------------|--------------|
| **Qwen-Turbo** | до 1 000 000 | 16 384 | Быстрые простые задачи |
| **Qwen-Plus** | до 1 000 000 | 32 768 | Повседневная разработка |
| **Qwen-Max** | 32 768 | 32 768 | Сложные архитектурные задачи |
| **Qwen-Long** | до 10 000 000 | 8 192 | Анализ больших документов |
| **Qwen-Flash** | до 1 000 000 | 16 384 | С кэшированием контекста |

---

### Что считается в лимите?

**Входные токены (input):**
- Ваши сообщения
- История предыдущих сообщений
- Содержимое файлов, открытых через `@`
- Результаты выполненных инструментов
- Системные промпты (частично)

**Выходные токены (output):**
- Ответы Qwen
- Сгенерированный код
- Объяснения и комментарии

**Общий лимит:**
```
Входные токены + Выходные токены ≤ Лимит модели
```

## Иерархия контекстных файлов

Qwen Code использует **контекстные файлы** для передачи инструкций модели.

### Уровни иерархии:

```
┌─────────────────────────────────────────┐
│  1. Глобальный контекст                 │
│     ~/.qwen/QWEN.md                     │
│     (инструкции для всех проектов)      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  2. Контекст проекта                    │
│     <проект>/QWEN.md                    │
│     (инструкции для конкретного проекта)│
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  3. Локальный контекст                  │
│     <подпапка>/QWEN.md                  │
│     (специфичные инструкции для папки)  │
└─────────────────────────────────────────┘
```

---

### Глобальный контекст

**Файл:** `~/.qwen/QWEN.md` (или `~/.qwen/instructions.md`)

**Назначение:** Инструкции, применяемые ко всем проектам.

**Пример содержимого:**
```markdown
# Глобальные инструкции

## Стиль общения
- Отвечай на русском языке
- Будь краток и конкретен
- Избегай излишних объяснений

## Предпочтения
- Используй Python для скриптов
- Предпочитай TypeScript для веба
- Всегда добавляй типы в код

## Запреты
- Не создавай файлы без подтверждения
- Не удаляй файлы без явного запроса
```

---

### Контекст проекта

**Файл:** `<проект>/QWEN.md`

**Назначение:** Инструкции для конкретного проекта.

**Пример для проекта "Собачья мудрость":**
```markdown
# Проект: Собачья мудрость. Секрет счастья

## Миссия
Создавать душевный, вдохновляющий контент, который затрагивает эмоциональную сферу читателей.

## Принципы контента
- Любовь без условий
- Принятие без критики
- Верность себе и другим
- Простота восприятия мира
- Радость настоящего момента

## Персонажи
- **Смельчак** — храбрый лабрадор, серебристо-белый окрас
- **Генрих** — мудрая овчарка, классический окрас
- **Эльза** — мудрая мать Смельчака

## Стиль
- Тёплый, эмпатичный тон
- Простой и доступный язык
- Глубокий психологический подтекст
- Использование метафор собак
```

---

### Автоматическая загрузка

Qwen Code автоматически ищет контекстные файлы:
1. В текущей директории
2. В родительских директориях (до `.git` или домашней папки)
3. В глобальной папке `~/.qwen/`

**Поиск останавливается** при нахождении:
- Файла `.git` (корень git-репозитория)
- Домашней директории пользователя

## Управление памятью

### Команды управления памятью

| Команда | Назначение |
|---------|----------|
| `/memory refresh` | Принудительная перезагрузка контекстных файлов |
| `/memory show` | Показать загруженный контекст |
| `/memory add <текст>` | Добавить информацию в память |
| `/memory list` | Показать сохранённую информацию |
| `/memory clear` | Очистить память |
| `/memory remove <ID>` | Удалить элемент по ID |

---

### `/memory refresh`

**Назначение:** Принудительно сканирует и перезагружает все контекстные файлы.

**Когда использовать:**
- Изменили `QWEN.md` во время сессии
- Добавили новый контекстный файл
- Qwen не применяет инструкции из файла

**Пример:**
```
/memory refresh
```

**Результат:**
```
✅ Контекстные файлы перезагружены
📁 Найдено файлов: 2
  - ~/.qwen/QWEN.md (глобальный)
  - D:\Projects\dog-wisdom\QWEN.md (проект)
```

---

### `/memory show`

**Назначение:** Показывает объединённый загруженный контекст.

**Когда использовать:**
- Хотите проверить, какие инструкции загружены
- Отлаживаете проблемы с контекстом
- Нужно скопировать контекст для отчёта

**Пример:**
```
/memory show
```

**Результат:**
```
📋 Загруженный контекст:

=== Глобальный контекст (~/.qwen/QWEN.md) ===
# Глобальные инструкции
Отвечай на русском языке...

=== Контекст проекта (QWEN.md) ===
# Проект: Собачья мудрость
Миссия: Создавать душевный контент...
```

---

### `/memory add`

**Назначение:** Добавляет информацию в долгосрочную память.

**Пример:**
```
/memory add Проект использует Python 3.11 и Flask
```

**Результат:**
```
✅ Добавлено в память [ID: 3]
"Проект использует Python 3.11 и Flask"
```

## Команды работы с контекстом

### Сжатие истории

| Команда | Назначение |
|---------|----------|
| `/summary` | Создать сводку сессии |
| `/compress` | Заменить историю на сводку |

---

### `/summary`

**Назначение:** Создаёт структурированную сводку текущей сессии.

**Пример:**
```
/summary
```

**Результат:**
```
## Сводка сессии

### Выполненные задачи:
- Создан модуль авторизации
- Исправлена ошибка валидации
- Написаны тесты для auth.py

### Изменённые файлы:
- src/auth.py
- tests/test_auth.py
- src/validators.py

### Открытые вопросы:
- Требуется интеграция с OAuth
- Нужно обновить документацию
```

---

### `/compress`

**Назначение:** Заменяет длинную историю на сжатую сводку.

**Важно:**
- Использует сводку от `/summary`
- **Необратимо** — нельзя вернуть детали
- Освобождает ~60-80% токенов

**Пример:**
```
/compress
```

**Результат:**
```
✅ История сжата
📈 Освобождено токенов: 45,000
📊 Новое использование: 23,680 / 100,000 (23.7%)
```

## Оптимизация использования токенов

### Что тратит токены

| Действие | Примерный расход |
|----------|------------------|
| Открытие файла через `@` | 100-5000 токенов |
| Чтение директории `@src/` | 5000-50000 токенов |
| Запрос с кодом | 500-2000 токенов |
| Ответ Qwen с кодом | 200-3000 токенов |
| Результат `!git diff` | 100-1000 токенов |
| Длинный вывод команды | 500-10000 токенов |

---

### Что НЕ тратит токены сессии

| Элемент | Расположение |
|---------|--------------|
| Контекстные файлы | `~/.qwen/QWEN.md`, `<проект>/QWEN.md` |
| Субагенты | `~/.qwen/agents/` |
| Пользовательские команды | `~/.qwen/commands/` |
| Настройки | `~/.qwen/settings.json` |
| Глобальная память | Внутреннее хранилище |

---

### Стратегии экономии

**1. Используйте контекстные файлы**
```
❌ Каждый раз объяснять проект
✅ Создать QWEN.md с инструкциями
```

**2. Открывайте файлы точечно**
```
❌ @src/ (вся директория)
✅ @src/main.py (конкретный файл)
```

**3. Используйте субагентов**
```
❌ Длинное объяснение задачи
✅ /agents create testing-expert
```

**4. Сжимайте историю вовремя**
```
❌ Ждать заполнения контекста
✅ /compress при 70-80%
```

**5. Проверяйте статистику**
```
❌ Работать без мониторинга
✅ Регулярно /stats
```

## Настройки контекста

### Файл настроек

**Расположение:** `~/.qwen/settings.json`

**Пример:**
```json
{
  "model": {
    "generationConfig": {
      "contextWindowSize": 128000
    },
    "maxSessionTurns": -1,
    "chatCompression": {
      "contextPercentageThreshold": 0.7
    }
  },
  "context": {
    "fileName": "QWEN.md",
    "includeDirectories": [],
    "loadFromIncludeDirectories": false
  }
}
```

---

### Ключевые параметры

#### `contextWindowSize`

**Описание:** Размер контекстного окна в токенах.

**Значение по умолчанию:** Зависит от модели (обычно 128,000)

**Пример:**
```json
{
  "model": {
    "generationConfig": {
      "contextWindowSize": 256000
    }
  }
}
```

---

#### `contextPercentageThreshold`

**Описание:** Порог автоматического сжатия истории (0-1).

**Значение по умолчанию:** `0.7` (70%)

**Что делает:**
- При достижении 70% контекста предлагает сжатие
- `0` = отключить автоматическое сжатие
- `0.5` = сжатие при 50% заполнения

**Пример:**
```json
{
  "model": {
    "chatCompression": {
      "contextPercentageThreshold": 0.8
    }
  }
}
```

---

#### `maxSessionTurns`

**Описание:** Максимальное количество ходов (запросов) в сессии.

**Значение по умолчанию:** `-1` (без ограничений)

**Пример:**
```json
{
  "model": {
    "maxSessionTurns": 100
  }
}
```

---

### Фильтрация файлов

**Настройки:**
```json
{
  "context": {
    "fileFiltering": {
      "respectGitIgnore": true,
      "respectQwenIgnore": true,
      "enableRecursiveFileSearch": true,
      "enableFuzzySearch": true
    }
  }
}
```

**Параметры:**

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `respectGitIgnore` | Учитывать `.gitignore` | `true` |
| `respectQwenIgnore` | Учитывать `.qwenignore` | `true` |
| `enableRecursiveFileSearch` | Рекурсивный поиск | `true` |
| `enableFuzzySearch` | Нечёткий поиск | `true` |

---

### `.qwenignore`

**Назначение:** Исключение файлов и папок из поиска.

**Пример:**
```
# Исключить большие директории
node_modules/
build/
dist/
__pycache__/
.venv/

# Исключить большие файлы
*.log
*.tmp
*.bak

# Исключить бинарные файлы
*.exe
*.dll
*.so
*.png
*.jpg
```

### ✅ Мониторинг контекста

**Регулярные проверки:**
```bash
# Начало сессии
/stats

# После крупных изменений
/stats

# Перед важным запросом
/stats
```

**Пороги для действий:**
- **50%** — нормально, работайте дальше
- **70%** — рассмотрите `/compress`
- **85%** — срочно сожмите или начните новую сессию
- **95%** — риск потери данных, немедленно сожмите

## Решение проблем

### Контекст заполняется слишком быстро

**Причины:**
- Открываете большие директории через `@`
- Длинные выводы команд
- Много итераций в диалоге

**Решения:**
1. Используйте `@` точечно
2. Применяйте `/compress` чаще
3. Увеличьте `contextWindowSize` в настройках
4. Используйте субагентов для изоляции задач

---

### Qwen не применяет инструкции из QWEN.md

**Проверка:**
```
/memory show
```

**Решения:**
1. Убедитесь, что файл в правильной папке
2. Проверьте синтаксис Markdown
3. Выполните `/memory refresh`
4. Перезапустите сессию

---

### Медленный поиск файлов

**Причины:**
- Большие директории (node_modules, build)
- Много файлов в проекте
- Включён нечёткий поиск

**Решения:**
1. Создайте `.qwenignore`
2. Отключите нечёткий поиск:
   ```json
   {"enableFuzzySearch": false}
   ```
3. Отключите рекурсивный поиск (крайняя мера):
   ```json
   {"enableRecursiveFileSearch": false}
   ```

---

### Ошибка "Context window exceeded"

**Что делать:**
1. Немедленно выполните `/compress`
2. Или начните новую сессию (`/quit`, затем новый запуск)
3. Увеличьте `contextWindowSize` в настройках
4. В будущем сжимайте историю раньше

---

## Шпаргалка по командам

| Команда | Когда использовать |
|---------|-------------------|
| `/stats` | Регулярно, для мониторинга |
| `/summary` | Перед сжатием, для фиксации прогресса |
| `/compress` | При 70-80% заполнения контекста |
| `/memory show` | Для проверки загруженных инструкций |
| `/memory refresh` | После изменения QWEN.md |
| `/memory add` | Для сохранения важной информации |

---

## Ключевые выводы

1. **Контекстные файлы НЕ тратят токены сессии** — используйте их для инструкций
2. **`@` файлы тратят токены** — открывайте точечно
3. **Субагенты имеют свой контекст** — делегируйте задачи
4. **Мониторьте `/stats`** — не допускайте переполнения
5. **Сжимайте вовремя** — при 70-80% заполнения
6. **Исключайте лишнее** — через `.qwenignore`
