## Что такое субагенты?

**Субагенты** — это специализированные ИИ-ассистенты внутри Qwen Code, которые выполняют конкретные типы задач.

### Ключевые особенности:

| Особенность | Описание |
|------------|----------|
| 🎯 **Специализация** | Каждый агент настроен на конкретный тип задач (тесты, документация, ревью) |
| 📁 **Отдельный контекст** | Субагент поддерживает собственную историю разговора |
| 🛠️ **Контролируемые инструменты** | Можно ограничить доступ к определённым инструментам |
| 🤖 **Автономная работа** | Работают независимо до завершения задачи |
| 📊 **Видимость прогресса** | Обновления в реальном времени о ходе выполнения |

---

## Ключевые преимущества

| Преимущество | Описание |
|-------------|----------|
| **Специализация задач** | Агенты оптимизированы для конкретных рабочих процессов |
| **Изоляция контекста** | Работа субагента не смешивается с основным разговором |
| **Многоразовость** | Конфигурации сохраняются и используются в других проектах |
| **Контролируемый доступ** | Ограничение инструментов для безопасности |
| **Видимость прогресса** | Мониторинг выполнения в реальном времени |

## Команды управления

### Основные команды

```
/agents create    # Создание нового субагента через пошаговый мастер
/agents manage    # Интерактивное управление существующими субагентами
```

---

### `/agents create`

**Назначение:** Запускает пошаговый мастер создания нового субагента.

**Когда использовать:**
- Нужен специализированный помощник для конкретного типа задач
- Хотите автоматизировать повторяющийся рабочий процесс
- Нужен агент с особыми настройками инструментов

**Что делает:**
- Запускает интерактивный диалог
- Пошагово запрашивает параметры агента
- Создаёт файл конфигурации
- Сохраняет в нужную директорию

**Пример использования:**
```
/agents create
```

**Шаги мастера:**

1. **Название агента:**
   ```
   Введите имя агента (например, testing-expert):
   > testing-expert
   ```

2. **Описание:**
   ```
   Введите описание (когда и как использовать этого агента):
   > Пишет комплексные юнит-тесты и интеграционные тесты
   ```

3. **Выбор инструментов:**
   ```
   Выберите инструменты (отметьте галочками):
   ☑ read_file
   ☑ write_file
   ☑ read_many_files
   ☐ run_shell_command
   ```

4. **Системный промпт:**
   ```
   Введите системный промпт (инструкцию для агента):
   > Вы — специалист по тестированию...
   ```

5. **Сохранение:**
   ```
   Где сохранить агента?
   [1] Проект (.qwen/agents/)
   [2] Глобально (~/.qwen/agents/)
   ```

**Результат:**
```
✅ Субагент "testing-expert" создан!
📁 Файл: .qwen/agents/testing-expert.md
```

---

### `/agents manage`

**Назначение:** Открывает интерактивный диалог для управления существующими субагентами.

**Когда использовать:**
- Хотите просмотреть всех доступных агентов
- Нужно отредактировать конфигурацию агента
- Требуется удалить или отключить агента
- Хотите изменить доступные инструменты

**Что делает:**
- Показывает список всех субагентов
- Позволяет редактировать, удалять, включать/выключать
- Отображает источник каждого агента

**Пример использования:**
```
/agents manage
```

**Результат:**
```
📋 Управление субагентами

📁 Проектные агенты (.qwen/agents/):
  [1] testing-expert ✅
  [2] documentation-writer ✅

🌍 Глобальные агенты (~/.qwen/agents/):
  [3] code-reviewer ✅
  [4] python-expert ✅

🔌 Агенты расширений:
  [5] react-specialist (от React Tools Extension)

Выберите действие:
[1] Просмотреть агента
[2] Редактировать
[3] Удалить
[4] Включить/Выключить
[5] Открыть папку агентов
[6] Выход
```

## Создание субагента

### Пошаговый процесс

**Шаг 1: Запуск мастера**
```
/agents create
```

**Шаг 2: Базовая информация**

Мастер запросит:
- **Имя:** Уникальное имя (латиница, дефисы)
- **Описание:** Когда и как использовать
- **Инструменты:** Какие инструменты доступны

**Шаг 3: Системный промпт**

Основная инструкция, которая определяет:
- Экспертизу агента
- Пошаговый подход к задачам
- Стандарты вывода
- Ограничения и требования

**Шаг 4: Выбор места сохранения**

| Уровень | Путь | Приоритет |
|---------|------|-----------|
| **Проектный** | `.qwen/agents/` | Наивысший |
| **Пользовательский** | `~/.qwen/agents/` | Резервный |
| **Расширений** | `agents/` в директории расширения | Автоматически |

## Формат файла субагента

### Базовая структура

Файлы субагентов — это **Markdown с YAML frontmatter**.

**Пример:**
```markdown
---
name: testing-expert
description: Пишет комплексные юнит-тесты, интеграционные тесты и занимается тестовой автоматизацией
tools:
  - read_file
  - write_file
  - read_many_files
  - run_shell_command
---

Вы — специалист по тестированию, сфокусированный на создании высококачественных тестов.

Ваша экспертиза включает:
- Юнит-тестирование с мокированием и изоляцией
- Интеграционное тестирование
- TDD практики
- Идентификация граничных случаев

Для каждой задачи:
1. Проанализируйте структуру кода
2. Определите ключевую функциональность
3. Создайте комплексные тесты
4. Добавьте комментарии
```

---

### YAML Frontmatter

**Обязательные поля:**

| Поле | Описание | Пример |
|------|----------|--------|
| `name` | Уникальное имя агента | `testing-expert` |
| `description` | Когда и как использовать | `Пишет юнит-тесты...` |

**Опциональные поля:**

| Поле | Описание | Пример |
|------|----------|--------|
| `tools` | Список доступных инструментов | `read_file, write_file` |
| `version` | Версия агента | `1.0.0` |
| `author` | Автор агента | `Олег Сдвижков` |

**Пример полного frontmatter:**
```yaml
---
name: testing-expert
description: Пишет комплексные юнит-тесты и интеграционные тесты
tools:
  - read_file
  - write_file
  - read_many_files
  - run_shell_command
version: 1.0.0
author: Олег Сдвижков
---
```

```markdown
# Область компетенции

- Навык 1

- Стандарт 1

- Ограничение 1

```

## Использование субагентов

### Автоматическое делегирование

Qwen Code **автоматически** делегирует задачи субагентам на основе:
- Описания задачи в вашем запросе
- Поля `description` в конфигурации субагента
- Текущего контекста

**Совет:** Для более проактивного использования включайте фразы `"используй ПРОАКТИВНО"` или `"ДОЛЖЕН ИСПОЛЬЗОВАТЬСЯ"` в поле `description`.

**Пример:**
```
Пользователь: "Напиши тесты для модуля оплаты"
ИИ: "Я делегирую это вашему testing-expert субагенту"
[Делегирует задачу]
[Показывает прогресс]
[Возвращает результат]
```

---

### Явный вызов

Можно явно указать, какого агента использовать:

**Примеры:**
```
Пусть testing-expert субагент создаст тесты для модуля платежей
Попроси documentation-writer обновить API документацию
Заставь code-reviewer проверить этот код на уязвимости
```

---

### Просмотр прогресса

Во время работы субагента вы видите:
- Текущий шаг выполнения
- Используемые инструменты
- Промежуточные результаты
- Статистику (время, токены)

**Пример:**
```
🤔 testing-expert анализирует код...
📖 read_file: src/payment.py ✅
📝 write_file: tests/test_payment.py (в процессе)
📊 Покрытие: 85%
✅ Завершено за 2.3 сек
```

## Безопасность

### 🔒 Соображения

| Аспект | Описание |
|--------|----------|
| **Ограничение инструментов** | Субагенты имеют доступ только к настроенным инструментам |
| **Песочница** | Выполнение следует той же модели безопасности |
| **Аудит-трейл** | Все действия логируются и видны |
| **Контроль доступа** | Разделение проектного и глобального уровня |
| **Чувствительная информация** | Не включайте секреты в конфигурации |

---

### ⚠️ Важные предупреждения

1. **Не включайте секреты:**
   ```markdown
   ❌ API_KEY = "sk-12345..."
   ✅ Используйте переменные окружения
   ```

2. **Ограничивайте опасные инструменты:**
   ```yaml
   # Для агентов документации не нужен доступ к командам
   tools:
     - read_file
     - write_file
     # run_shell_command — не включён
   ```

3. **Разделяйте окружения:**
   - Отдельные агенты для development
   - Отдельные агенты для production

## Субагенты расширений

**Расширения** могут предоставлять собственные субагенты.

**Особенности:**
- Автоматически обнаруживаются
- Появляются в `/agents manage` в секции "Extension Agents"
- Нельзя редактировать напрямую
- Следуют тому же формату

**Примеры расширений:**
- React Tools Extension → react-specialist
- Python Pro Extension → python-debugger
- Security Pack → security-auditor

## Решение проблем

### Агент не появляется в списке

**Проверка:**
1. Файл в правильной папке?
2. YAML frontmatter корректен?
3. Имя уникально?
4. Перезапустили Qwen Code?

**Решение:**
```bash
# Проверка структуры
ls .qwen/agents/
cat .qwen/agents/agent-name.md
```

---

### Агент не выполняет задачи правильно

**Возможные причины:**
1. Нечёткий системный промпт
2. Недостаточно инструментов
3. Конфликт с другим агентом

**Решение:**
```
/agents manage
# Выберите агента → Редактировать
# Уточните промпт
# Добавьте инструменты
```

---

### Конфликт имён

**Проблема:** Два агента с одинаковым именем.

**Решение:**
1. Проектные агенты имеют приоритет
2. Переименуйте одного из агентов
3. Используйте префиксы:
   ```
   project-testing-expert.md
   global-testing-expert.md
   ```

## Шаблон для создания агента

**Файл:** `TEMPLATE.md`

```markdown
---
name: agent-name
description: Краткое описание, когда и как использовать
tools:
  - read_file
  - write_file
version: 1.0.0
author: Ваше имя
---

# Роль
Вы — [специализация] с экспертизой в [область].

# Экспертиза
- Навык 1
- Навык 2
- Навык 3

# Пошаговый подход
1. Шаг 1
2. Шаг 2
3. Шаг 3

# Стандарты
- Стандарт 1
- Стандарт 2

# Ограничения
- Не делайте X
- Избегайте Y
```
