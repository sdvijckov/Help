# 📋 Копирование, Вставка и Опыт работы с мастером агентов

В Qwen Code очень интресная система вставки текста в чат. Она заключается в том, что:
1. в строке чата нужно нажать ctrl+x
2. откроется файл блокнонота Buffer
3. в этот файл нужно вставить текст и закрыть с сохранением
4. текст уже будет в поле
Так же можно копировать уже написанное в поле 

## 2. Создание субагента через `/agents create`

Теперь расскажу о своём опыте работы с /agents
**Первый раз было так:**

1. спросил уровень на котором сохранить подагента: проектный или пользовательский. Я выбрал проектный. Он добавил папку .qwen в проекте. Наверное, если бы я выбрал пользовательский, он бы сделал папку agents в глобальной папке и он бы стал доступен всем проектам
2. спросил сгенерировать профиль или добавить вручную. Я выбрал сгенерироваать
3. попросил описать агента. я написал: Преподаватель английского языка работает в паре с преподавателем русского языка
4. выбрал инструменты. он сгенерировал профиль и агент появился в /agents manage

**Второй раз:**

1. я выбрал пользовательский
2. ручное создание
3. имя агента: Smelchak_English_Teacher
4. системный промпт из инструкции для Смельчака
5. описание: Учит английскому яязыку. Smelchak_English_Teacher работает в паре с Genrikh_Russian_Teacher
6. инструменты; все инструменты
7. цвет фона
8. подтверждение

### Пошаговый процесс (на примере Смельчака):

**Шаг 1: Запуск мастера**
```
/agents create
```

**Шаг 2: Ввод имени агента**
```
Мастер: Введите имя агента (например, testing-expert):
Олег: smelchak-english-teacher
```

**Шаг 3: Ввод описания**
```
Мастер: Введите описание (когда и как использовать этого агента):
Олег: Преподаватель английского языка, работает в паре с преподавателем русского
```

**Шаг 4: Выбор инструментов**
```
Мастер: Выберите инструменты (отметьте галочками):
☑ read_file
☑ write_file
☑ web_search
☐ run_shell_command
```

**Шаг 5: Генерация или ручной ввод**
```
Мастер: Сгенерировать профиль или добавить вручную?
Олег: Сгенерировать
```

**Шаг 6: Мастер сгенерировал профиль**
- Qwen Code создал профиль на основе описания
- Профиль включал YAML frontmatter и системный промпт

**Шаг 7: Выбор уровня сохранения**
```
Мастер: Где сохранить агента?
[1] Проект (.qwen/agents/)
[2] Глобально (~/.qwen/agents/)
Олег: [2] Глобально
```

**Результат:**
```
✅ Субагент "smelchak-english-teacher" создан!
📁 Файл: ~/.qwen/agents/smelchak-english-teacher.md
```

## 6. Тестирование

### Проверка чтения файлов:
```
Смельчак, прочитай раздел VII из ~/.qwen/dog-wisdom-agents/Smelchak_Full_Profile.md

Генрих, прочитай сценарий 1 из ~/.qwen/dog-wisdom-agents/Duo_Teaching_Guide.md
```

### Проверка взаимодействия:
```
Проведите совместный урок на тему «Верность»
```

### Ожидаемый результат:
- ✅ Смельчак начинает с энергичного приветствия
- ✅ Генрих добавляет глубину
- ✅ Используют цитаты из полных профилей
- ✅ Передают эстафету друг другу

## 7. Заметки на будущее

### Что сработало:
- ✅ Мастер `/agents create` быстро создаёт базовый профиль
- ✅ Замена на готовый промпт занимает 2-3 минуты
- ✅ Поле `documentation` в YAML позволяет субагентам знать о полных профилях

### Что можно улучшить:
- [ ] Автоматическое создание поля `documentation` мастером
- [ ] Возможность указания путей к внешним файлам в мастере
- [ ] Экспорт/импорт профилей между проектами

### Советы новичкам:
1. **Не полагайтесь на генерацию** — мастер не знает контекста вашего проекта
2. **Готовьте системные промпты заранее** — в отдельных файлах
3. **Используйте поле `documentation`** — для связи с полными профилями
4. **Тестируйте чтение файлов** — убедитесь, что субагенты могут читать документацию
