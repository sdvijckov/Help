# 07_File-команды (@)

> Быстрое добавление содержимого файлов в разговор с помощью @

---

## Что такое File-команды?

File-команды — это способ быстро добавить содержимое файлов или директорий в контекст разговора с Qwen Code, используя символ `@`.

**Синтаксис:**
```
@<путь к файлу или директории>
```

**Пример:**
```
Посмотри этот файл: @src/main.py
```

---

## Основные команды

| Синтаксис | Назначение | Пример |
|-----------|----------|--------|
| `@<файл>` | Вставить содержимое одного файла | `@src/main.py` |
| `@<директория>` | Рекурсивно прочитать все текстовые файлы в директории | `@docs/` |
| `@<несколько файлов>` | Добавить несколько файлов сразу | `@file1.py @file2.py` |

---

## Использование с файлами

### Чтение одного файла

**Синтаксис:**
```
@<относительный_путь/к/файлу>
```

**Примеры:**

```
Объясни, как работает этот код: @src/utils/helpers.py
```

```
Найди ошибки в: @config/settings.json
```

```
Проверь тесты: @tests/test_auth.py
```

**Что происходит:**
1. Qwen читает содержимое файла
2. Добавляет его в контекст разговора
3. Анализирует и отвечает на ваш запрос

**Результат:**
```
📄 Прочитан файл: src/utils/helpers.py (156 строк)

[Содержимое файла добавлено в контекст]

Теперь я могу ответить на ваш вопрос о коде...
```

---

### Чтение нескольких файлов

**Синтаксис:**
```
@<файл1> @<файл2> @<файл3>
```

**Примеры:**

```
Сравни реализацию: @src/auth_v1.py @src/auth_v2.py
```

```
Проверь согласованность: @frontend/app.js @backend/app.py
```

```
Объедини эти файлы: @part1.md @part2.md @part3.md
```

**Что происходит:**
- Qwen читает все указанные файлы
- Добавляет каждый в контекст
- Может сравнивать, анализировать, объединять

---

### Пути к файлам

**Относительные пути** (рекомендуется):
```
@src/main.py
@../config.json
@./tests/test.py
@../../shared/utils.py
```

**Абсолютные пути** (Windows):
```
@D:\Projects\myapp\src\main.py
```

**Абсолютные пути** (Linux/Mac):
```
@/home/user/projects/myapp/src/main.py
```

**С пробелами в пути** (используйте кавычки):
```
@"my project/src/main.py"
```

## Использование с директориями

### Чтение всей директории

**Синтаксис:**
```
@<путь/к/директории>/
```

**Важно:** Добавьте `/` в конце, чтобы указать, что это директория.

**Примеры:**

```
Проанализируй структуру проекта: @src/
```

```
Найди все проблемы в коде: @tests/
```

```
Сделай ревью: @docs/
```

**Что происходит:**
1. Qwen сканирует директорию
2. Находит все текстовые файлы
3. Пропускает бинарные файлы и исключения
4. Добавляет содержимое в контекст

**Результат:**
```
📁 Прочитана директория: src/

Найдено файлов: 12
- src/main.py (245 строк)
- src/utils.py (89 строк)
- src/auth.py (156 строк)
- src/config.py (34 строки)
...

Общий размер: 1,234 строки

[Содержимое добавлено в контекст]
```

---

### Исключения при чтении директорий

Qwen автоматически пропускает:

**Системные папки:**
- `.git/`
- `node_modules/`
- `__pycache__/`
- `.venv/`, `venv/`
- `dist/`, `build/`

**Бинарные файлы:**
- `.exe`, `.dll`, `.so`
- `.png`, `.jpg`, `.gif`
- `.pdf`, `.docx`
- `.zip`, `.tar`, `.gz`

**Файлы из `.gitignore`:**
- Учитываются правила игнорирования

**Большие файлы:**
- Файлы больше определённого лимита (обычно > 100KB)

## Поддерживаемые форматы файлов

### ✅ Текстовые файлы (поддерживаются)

**Код:**
- `.py`, `.js`, `.ts`, `.jsx`, `.tsx`
- `.java`, `.cpp`, `.c`, `.h`, `.cs`
- `.go`, `.rs`, `.php`, `.rb`, `.swift`
- `.kt`, `.scala`, `.r`, `.m`

**Веб:**
- `.html`, `.css`, `.scss`, `.less`
- `.vue`, `.svelte`

**Конфигурация:**
- `.json`, `.yaml`, `.yml`, `.toml`
- `.xml`, `.ini`, `.cfg`, `.conf`
- `.env`, `.env.example`

**Документация:**
- `.md`, `.txt`, `.rst`
- `.adoc`, `.org`

**Скрипты:**
- `.sh`, `.bash`, `.bat`, `.cmd`
- `.ps1`, `.fish`

**Другие:**
- `.sql`, `.graphql`
- `.proto`, `.thrift`
- `.dockerfile`, `Dockerfile`
- `Makefile`, `CMakeLists.txt`

---

### ❌ Бинарные файлы (не поддерживаются)

**Изображения:**
- `.png`, `.jpg`, `.jpeg`, `.gif`
- `.bmp`, `.tiff`, `.webp`, `.svg`

**Документы:**
- `.pdf`, `.docx`, `.xlsx`, `.pptx`

**Архивы:**
- `.zip`, `.rar`, `.7z`, `.tar`, `.gz`

**Исполняемые файлы:**
- `.exe`, `.dll`, `.so`, `.bin`

**Медиа:**
- `.mp3`, `.mp4`, `.avi`, `.mov`

**Примечание:** Для работы с некоторыми бинарными файлами используйте навыки (`/skills`).

## Советы по использованию

### ✅ Лучшие практики

1. **Используйте относительные пути** — они короче и переносимее.
   ```
   @src/main.py  ✅
   @D:\Projects\...\src\main.py  ❌
   ```

2. **Группируйте связанные файлы** — для контекстного анализа.
   ```
   @src/auth.py @tests/test_auth.py
   ```

3. **Читайте директории для обзора** — чтобы Qwen понял структуру.
   ```
   @src/
   ```

4. **Избегайте больших директорий** — может превысить лимит токенов.
   ```
   @node_modules/  ❌
   @src/  ✅
   ```

5. **Проверяйте, что читаете** — убедитесь, что файл существует.
   ```
   @src/mainn.py  ❌ (опечатка)
   ```

---

### ⚠️ Ограничения

**Лимит токенов:**
- Один файл не должен превышать ~50,000 токенов
- Общая сумма файлов в запросе учитывается в лимите

**Лимит файлов:**
- Максимум ~50 файлов за один запрос
- Для больших проектов читайте по частям

**Время чтения:**
- Большие директории могут читаться несколько секунд
- Qwen покажет прогресс

---

### 🔧 Комбинация с другими командами

**С `/init`:**
```

```

```

```


## Решение проблем

### Файл не найден

**Ошибка:**
```
❌ Файл не найден: @src/mainn.py
```

**Решение:**
1. Проверьте путь
2. Используйте `!ls` или `!dir` для проверки
3. Убедитесь, что файл существует

```
!ls src/
@src/main.py  # Исправленный путь
```

---

### Файл слишком большой

**Ошибка:**
```
⚠️ Файл слишком большой: @data/large_dataset.json (2.5 MB)
```

**Решение:**
1. Откройте файл частями
2. Используйте текстовый редактор
3. Извлеките нужную часть

```
# Прочитать только первые 100 строк
@data/large_dataset.json (строки 1-100)
```

---

### Директория пуста

**Предупреждение:**
```
⚠️ Директория пуста или содержит только игнорируемые файлы: @temp/
```

**Решение:**
- Проверьте, что файлы не в `.gitignore`
- Убедитесь, что файлы текстовые
- Проверьте путь к директории
