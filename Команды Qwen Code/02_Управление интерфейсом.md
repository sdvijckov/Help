# 02_Управление интерфейсом

> Команды для настройки внешнего вида и рабочей среды Qwen Code

---

## Список команд

| Команда | Назначение |
|---------|----------|
| [`/clear`](#clear) | Очистка содержимого терминала |
| [`/theme`](#theme) | Смена визуальной темы |
| [`/vim`](#vim) | Включение режима редактирования Vim |
| [`/directory`](#directory-или-dir) | Управление рабочими директориями |
| [`/editor`](#editor) | Выбор внешнего редактора кода |

---

## `/clear`

**Назначение:** Очищает содержимое терминала, удаляя весь видимый текст из текущего окна сессии.

**Когда использовать:**
- Хочется «освежить» рабочий экран
- Завершили этап работы и начинаете новый
- Терминал переполнен выводом и стало неудобно читать
- Нужно быстро очистить экран без завершения сессии

**Что делает:**
- Удаляет весь видимый текст из окна терминала
- **Не удаляет** историю переписки — Qwen всё помнит
- Очищает только визуальную часть

**Пример:**
```
/clear
```

**Горячая клавиша:** `Ctrl+L` (или `Cmd+L` на Mac) — полный аналог команды.

**Важно:** История сессии сохраняется! Очищается только экран. Для сброса истории используйте `/compress`.

## `/theme`

**Назначение:** Открывает диалог для выбора визуальной темы интерфейса Qwen Code.

**Когда использовать:**
- Хочется сменить цветовую схему
- Устали глаза от текущей темы
- Работаете в разное время суток (день/ночь)
- Предпочитаете определённый стиль оформления

**Что делает:**
- Показывает список доступных тем
- Позволяет выбрать между светлой и тёмной темой
- Применяет выбранную тему немедленно

**Пример:**
```
/theme
```

**Типичные варианты тем:**
- **Dark** — тёмная тема (классика для разработчиков)
- **Light** — светлая тема (лучше для дневной работы)
- **System** — следует системным настройкам ОС

**Совет:** Тёмная тема снижает нагрузку на глаза при работе в условиях низкой освещённости.

## `/vim`

**Назначение:** Включает или выключает режим редактирования Vim в области ввода команд.

**Когда использовать:**
- Вы привыкли к Vim/Neovim
- Хотите использовать Vim-команды для навигации по тексту
- Нужно эффективно редактировать длинные промпты
- Предпочитаете modal editing (режимы вставки и навигации)

**Что делает:**
- Включает Vim-режим в поле ввода
- Позволяет использовать команды: `h`, `j`, `k`, `l` для навигации
- Добавляет режимы: Normal, Insert, Visual
- Переключает состояние (вкл/выкл) при каждом вызове

**Пример:**
```
/vim
```

**Базовые команды Vim в режиме Normal:**

| Клавиша | Действие |
|---------|----------|
| `h` `j` `k` `l` | Перемещение курсора |
| `w` `b` | Переход по словам |
| `0` `$` | В начало/конец строки |
| `i` | Режим вставки |
| `Esc` | Выход в Normal mode |
| `dd` | Удалить строку |
| `yy` | Копировать строку |
| `p` | Вставить |

**Важно:** Если вы не знакомы с Vim, эта команда может усложнить работу. Используйте только если уже умеете работать в Vim.

## `/directory` (или `/dir`)

**Назначение:** Управляет поддержкой нескольких директорий в одной сессии. Позволяет добавлять и удалять рабочие папки.

**Когда использовать:**
- Работаете с проектом, состоящим из нескольких папок
- Нужно дать Qwen доступ к разным частям проекта
- Разрабатываете несколько связанных проектов одновременно
- Хотите переключаться между контекстами

**Синтаксис:**
```
/dir add <путь1>,<путь2>    # Добавить директории
/dir remove <путь>          # Удалить директорию
/dir list                   # Показать список директорий
```

**Примеры:**
```
/dir add ./src,./tests,./docs
# Добавит три директории для работы

/dir list
# Покажет:
# - D:\Projects python\Help
# - D:\Projects python\Help\src
# - D:\Projects python\Help\tests

/dir remove ./tests
# Удалит директорию tests из рабочей области
```

**Что делает:**
- Расширяет рабочую область Qwen
- Даёт доступ к файлам в указанных папках
- Позволяет использовать `@` с файлами из всех добавленных директорий

**Важно:** Все пути указываются относительно текущей рабочей директории.

## `/editor`

**Назначение:** Открывает диалог для выбора внешнего редактора кода, который будет использоваться для редактирования файлов.

**Когда использовать:**
- Хотите редактировать файлы в любимом редакторе
- Qwen использует не тот редактор по умолчанию
- Установили новый редактор и хотите его подключить
- Нужно переключиться между разными редакторами

**Что делает:**
- Показывает список поддерживаемых редакторов
- Позволяет выбрать редактор по умолчанию
- Настраивает интеграцию с выбранным редактором

**Пример:**
```
/editor
```

**Поддерживаемые редакторы (типичные):**
- **VS Code** — самый популярный выбор
- **Vim / Neovim** — для любителей терминала
- **Sublime Text** — лёгкий и быстрый
- **Notepad++** — простой текстовый редактор
- **JetBrains IDE** (PyCharm, WebStorm, etc.)

**После выбора:**
- Файлы будут открываться в выбранном редакторе
- Изменения, сделанные в редакторе, будут видны Qwen
- Qwen сможет предлагать правки с учётом возможностей редактора

**Совет:** Для максимальной интеграции рекомендуется VS Code с установленным расширением Qwen Code (если доступно).
