# 05_Режимы подтверждения инструментов

> Команды для настройки уровня подтверждения при использовании инструментов Qwen Code

---

## Список команд

| Команда | Назначение | Уровень безопасности |
|---------|----------|---------------------|
| [`/approval-mode plan`](#approval-mode-plan) | Только анализ, без выполнения | 🔒 Максимальный |
| [`/approval-mode default`](#approval-mode-default) | Подтверждение для редактирования | 🛡️ Стандартный |
| [`/approval-mode auto-edit`](#approval-mode-auto-edit) | Авто-подтверждение редактирования | ⚡ Доверенный |
| [`/approval-mode yolo`](#approval-mode-yolo) | Полное авто-подтверждение | 🚀 Без ограничений |

---

## Зачем нужны режимы подтверждения?

Qwen Code может выполнять действия в вашей системе:
- ✏️ Редактировать файлы
- 🗑️ Удалять файлы
- 📁 Создавать папки
- 🖥️ Выполнять команды в терминале
- 🌐 Делать запросы в интернет

**Режимы подтверждения** определяют, насколько строго Qwen должен спрашивать разрешение перед каждым действием.

---

## `/approval-mode plan`

**Назначение:** Режим только для анализа. Qwen показывает, что он *хотел бы* сделать, но не выполняет никаких действий.

**Когда использовать:**
- 🔍 Изучаете чужой код и хотите понять, что можно изменить
- 📚 Обучаетесь и хотите видеть план действий
- 🧪 Тестируете сложные задачи перед запуском
- ⚠️ Работаете с критически важным кодом (продакшен, база данных)
- 💰 Оцениваете объём работы перед запуском

**Что делает:**
- Анализирует ваш запрос
- Показывает подробный план действий
- **Не выполняет** никаких изменений
- Ждёт дальнейших инструкций

**Пример:**
```
/approval-mode plan
```

**Запрос после включения:**
```
Пользователь: Удали все файлы в папке temp/

Qwen: [ПЛАН]
Я планирую выполнить следующие действия:

1. list_directory("temp/")
   → Показать содержимое папки

2. run_shell_command("rm -rf temp/*")
   → Удалить все файлы в папке

⚠️ Это действие необратимо! Продолжить?
[1] Выполнить план
[2] Изменить план
[3] Отмена
```

**Уровень безопасности:** 🔒 **Максимальный**

**Рекомендация:** Используйте для первого знакомства с Qwen Code или при работе с незнакомым кодом.

---

## `/approval-mode default`

**Назначение:** Стандартный режим. Требует подтверждения для операций редактирования и выполнения команд.

**Когда использовать:**
- 📝 Повседневная разработка
- ✅ Стандартная работа с кодом
- 🛡️ Баланс между безопасностью и удобством
- 👥 Работа в команде (рекомендуется для всех участников)

**Что делает:**
- Автоматически разрешает **чтение** файлов
- Требует подтверждения для **записи/изменения** файлов
- Требует подтверждения для **выполнения команд**
- Показывает краткое описание каждого действия

**Пример:**
```
/approval-mode default
```

**Запрос после включения:**
```
Пользователь: Создай файл test.py с функцией hello

Qwen: [ТРЕБУЕТСЯ ПОДТВЕРЖДЕНИЕ]

📝 write_file("test.py")
   Создание нового файла с содержимым:
   
   def hello():
       print("Hello, World!")

[1] Разрешить
[2] Отклонить
[3] Показать дифф
```

**Уровень безопасности:** 🛡️ **Стандартный**

**Рекомендация:** **Режим по умолчанию для ежедневной работы.** Лучший баланс безопасности и продуктивности.

---

## `/approval-mode auto-edit`

**Назначение:** Автоматически подтверждает операции редактирования файлов, но требует подтверждения для команд.

**Когда использовать:**
- ⚡ Быстрая итеративная разработка
- 🏃 Прототипирование и эксперименты
- ✅ Доверяете Qwen и хотите ускорить работу
- 🔁 Много мелких правок кода

**Что делает:**
- Автоматически разрешает **чтение и запись** файлов
- Требует подтверждения для **выполнения команд**
- Требует подтверждения для **удаления файлов**
- Показывает уведомления о выполненных действиях

**Пример:**
```
/approval-mode auto-edit
```

**Запрос после включения:**
```
Пользователь: Добавь импорт os в начало файла main.py

Qwen: ✅ Изменено: main.py
   + Добавлена строка: import os
   + Позиция: строка 1

---

Пользователь: Запусти тесты

Qwen: [ТРЕБУЕТСЯ ПОДТВЕРЖДЕНИЕ]

🖥️ run_shell_command("pytest tests/")
   Выполнение команды в терминале

[1] Разрешить
[2] Отклонить
```

**Уровень безопасности:** ⚡ **Доверенный**

**Рекомендация:** Используйте в личных проектах или когда полностью доверяете Qwen.

---

## `/approval-mode yolo`

**Назначение:** Полное автоматическое подтверждение всех действий. **Используйте с осторожностью!**

> ⚠️ **YOLO** = "You Only Live Once" (живём один раз). Режим назван иронично — он самый рискованный.

**Когда использовать:**
- 🚀 Быстрое прототипирование
- 🧪 Изолированная среда (песочница, Docker)
- 🤖 Автоматизированные пайплайны
- ⚡ Демонстрации и презентации
- 💾 Есть свежий бэкап или Git

**Когда НЕ использовать:**
- ❌ Работа с продакшен-кодом
- ❌ Работа с базами данных
- ❌ Работа с важными файлами без бэкапа
- ❌ Непроверенные или сложные запросы

**Что делает:**
- Автоматически подтверждает **все действия**
- Не показывает диалоги подтверждения
- Выполняет запросы максимально быстро
- **Вы несёте полную ответственность за результат**

**Пример:**
```
/approval-mode yolo
```

**Запрос после включения:**
```
Пользователь: Перепиши функцию calculate() с использованием кэширования

Qwen: ✅ Изменено: utils.py
   ✅ Функция calculate() обновлена
   ✅ Добавлен декоратор @lru_cache
   ✅ Выполнено за 0.3 сек

---

Пользователь: Запусти бенчмарки

Qwen: ✅ Выполнено: python benchmark.py
   📊 Результаты:
   - До: 150 ops/sec
   - После: 450 ops/sec (+200%)
```

**Уровень безопасности:** 🚀 **Без ограничений** (опасно!)

**Предупреждение:**
> ⚠️ **Внимание!** В этом режиме Qwen может:
> - Удалить важные файлы
> - Выполнить опасные команды
> - Внести нежелательные изменения
> 
> **Используйте только в контролируемой среде!**

## Сравнение режимов

| Действие | `plan` | `default` | `auto-edit` | `yolo` |
|----------|--------|-----------|-------------|--------|
| Чтение файлов | ✅ План | ✅ Авто | ✅ Авто | ✅ Авто |
| Запись файлов | ✅ План | ⚠️ Вопрос | ✅ Авто | ✅ Авто |
| Удаление файлов | ✅ План | ⚠️ Вопрос | ⚠️ Вопрос | ✅ Авто |
| Выполнение команд | ✅ План | ⚠️ Вопрос | ⚠️ Вопрос | ✅ Авто |
| Web-запросы | ✅ План | ⚠️ Вопрос | ⚠️ Вопрос | ✅ Авто |
| Использование субагентов | ✅ План | ⚠️ Вопрос | ⚠️ Вопрос | ✅ Авто |

**Условные обозначения:**
- ✅ **Авто** — выполняется автоматически
- ⚠️ **Вопрос** — требует подтверждения

## Как выбрать режим?

### 🎯 Сценарии использования

**1. Новый проект / незнакомый код**
```
/approval-mode plan
```
*Изучите код и план действий перед внесением изменений.*

**2. Повседневная разработка**
```
/approval-mode default
```
*Стандартный режим для большинства задач.*

**3. Быстрое прототипирование (личный проект)**
```
/approval-mode auto-edit
```
*Ускоряет работу при доверии к Qwen.*

**4. Песочница / Docker / тестовая среда**
```
/approval-mode yolo
```
*Максимальная скорость, риск минимален.*

**5. Работа с продакшеном**
```
/approval-mode plan
# или
/approval-mode default
```
*Максимальная осторожность!*

---

## Переключение режимов в процессе работы

Вы можете менять режимы в любой момент:

```
# Начало работы — изучение
/approval-mode plan

# После утверждения плана — стандартная работа
/approval-mode default

# Фаза активного кодирования
/approval-mode auto-edit

# Финальное тестирование — осторожно
/approval-mode default
```

---

## Проверка текущего режима

Чтобы узнать текущий режим:

```
/stats
```

Или просто обратите внимание на поведение Qwen:
- Показывает план без выполнения → `plan`
- Спрашивает подтверждение для правок → `default`
- Правит файлы без спроса, спрашивает для команд → `auto-edit`
- Никаких вопросов → `yolo`

---

## Советы по безопасности

1. **Всегда делайте коммиты в Git** перед запуском Qwen в режимах `auto-edit` или `yolo`.

2. **Используйте `/approval-mode plan`** для сложных или незнакомых задач.

3. **Не используйте `yolo`** в рабочих проектах без бэкапа.

4. **Переключайтесь на `default`** перед завершением сессии.

5. **Настройте `.gitignore`** для защиты важных файлов.

6. **Используйте песочницу** (Docker, VM) для экспериментов в режиме `yolo`.
